Полная документация: https://docs.onerpa.ru/mcp-servery-1c

Для работы с MCP серверами ОБЯЗАТЕЛЬНО 
включите хотя бы часть про инструменты из:
https://github.com/comol/cursor_rules_1c
в Project_rules 
Если не понимаете о чём речь - https://vibecoding1c.ru/#course крайне важен

Требования:
- Windows 10 (версия 2004+) или Windows 11
- Docker Desktop с WSL2
- Cursor IDE или другой клиент с поддержкой MCP
- 8+ ГБ RAM (рекомендуется 16 ГБ)
- SSD для векторных баз данных

ВАЖНО: Обязательно монтируйте тома (-v) для сохранения векторных баз данных!
Индексация может занимать от нескольких часов до суток. Без монтирования томов 
все индексы будут потеряны при перезапуске контейнера.
Подробнее: https://docs.onerpa.ru/mcp-servery-1c/prodvinutoe-ispolzovanie/keshirovanie-bd

Embedding модели:
- Рекомендуется LM Studio + Qwen3 Embedding (если есть GPU NVIDIA) - лучшее качество и скорость
- CPU режим работает везде, но медленнее
- Для пользователей из России: при использовании CPU-моделей требуется скачивание с huggingface.co,
  который может быть заблокирован. Рекомендуем использовать LM Studio.
Подробнее: https://docs.onerpa.ru/mcp-servery-1c/embedding-modeli

Файлы в порядке убывания важности MCP:
HelpSearchServer - Справка по платформе. Именно вашей версии. Крайне важно.
Graph_metadata_search - Графовый поиск по метаданным. При прочих равных лучше использовать его
CodeMetadataSearchServer - Поиск по метаданным, поиск по справке конфигурации, поиск паттернов по коду конфигурации
SSLSearchServer - если у вас используется БСП этот MCP просто незаменим. Поиск по справке к БСП нужной вам версии
TemplatesSearchServer - Поиск по публично доступным шаблонам кода 1С. в сотни раз эффективнее чем Web поиск + можно и НУЖНО добавлять свои шаблоны
SyntaxCheckServer - проверка синтаксиса через BSL LS
1CCodeChecker - проверка синтаксиса и логики кода через 1С напарника
