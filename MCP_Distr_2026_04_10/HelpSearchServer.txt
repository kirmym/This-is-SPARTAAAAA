Инструкция по установке:

1) В случае с Windows нужен Docker Desktop - скачиваем и устанавливаем. И WSL2. Под Linux и Mac нужен просто Docker

2) Запускаем команду:

docker run -d `
-e RESET_CACHE=false `
-e RESET_DATABASE=false `
-e USESSE=false `
-e LICENSE_KEY=5a98417f-de9f-468c-b62c-d940a5681da8 `
-e 1C_BIN_PATH=/1c_docs `
-e OPENAI_API_BASE="https://openrouter.ai/api" `
-e OPENAI_API_KEY="мой ключ API" `
-e OPENAI_MODEL="qwen/qwen3-embedding-8b" `
-p 8003:8003 `
-v "C:\Program Files (x86)\1cv8\8.3.27.1936\bin:/1c_docs" `
-v "E:/bases/mcp_docs:/app/chroma_db" `
comol/1c_help_mcp:light

Самая универсальная команда запуска. Использует Light версию контейнера. В неё нужно поменять только путь к вашей bin папке платформы (C:\Program Files (x86)\1cv8\8.3.27.1936\bin), путь куда сохранять базу (E:/bases/mcp_docs)
и ключ к OpenRouter ("мой ключ API"). На OpenRouter нужно конечно зарегистрироваться.

Простейшая же команда будет следующая:
docker run -d -p 8003:8003 -e LICENSE_KEY=5a98417f-de9f-468c-b62c-d940a5681da8 -e 1C_BIN_PATH=/1c_docs -v "C:\Program Files (x86)\1cv8\8.3.27.1936\bin:/1c_docs" comol/1c_help_mcp:latest
Ваш лицензионный ключ уже вшит в команду 
"C:\Program Files (x86)\1cv8\8.3.27.1936\bin" - меняем на путь к вашей папке bin платформы 1С. Другие значения командной строки не меняем

У контейнера возможны метки :light, :lastest, :arm64. Первая использует только облачное API (ну или LMStudio), вторая - может загрузить локально модель, третья - для процессоров arm64


3) Ждём! Индексация может занять до нескольких часов. Посмотреть текущий статус можно через docker logs. Или "-d" в параметрах команды меняем на "-it" и не закрываем окно терминала.
При старте в систему будет выкачиваться контейнер и ещё модель сверху. Нужен доступ в Интернет и до 10ГБ на диске.

ВАЖНО: Обязательно используйте параметр -v для сохранения индексов! Без этого при перезапуске 
контейнера потребуется повторная индексация (несколько часов).

4) mcp.json для Cursor будет выглядеть вот так. Если серверов больше, они добавляются через запятую
{
  "mcpServers": {
    "1c-docs-mcp": {
      "url": "http://localhost:8003/mcp",
      "connection_id": "1c_docs_service_001"
    }
  }
} 

UPD более продвинутый запуск:

docker run -d -e RESET_CACHE=false -e RESET_DATABASE=false -e USESSE=false -e LICENSE_KEY=5a98417f-de9f-468c-b62c-d940a5681da8 -e 1C_BIN_PATH=/1c_docs -p 8003:8003 -v "C:\Program Files (x86)\1cv8\8.3.27.1936\bin:/1c_docs" -v "E:/bases/mcp_docs:/app/chroma_db" comol/1c_help_mcp:latest

По умолчанию RESET_CACHE=true RESET_DATABASE=true USESSE=false
RESET_CACHE - заново загружает модель в контейнер. Если false - будет использоваться кэшированная
RESET_DATABASE - полностью переииндиксировать все данные. Для этого надо базу обязательно
USESSE - в качестве транспорта использует SSE. Необходимо для Legacy клиентов. 
E:/bases/mcp_docs - путь до каталога где можно создать векторную БД

Можно использовать внешнюю локальную модель Embedding - для более правильной RAG индексации.
Для этого надо можно установить три параметра:
OPENAI_API_BASE
OPENAI_API_KEY
OPENAI_MODEL
Если их не устанавливать - будут использованы дефолтные значения, устанавливаемые после запуска LM Studio.
Модели можно использовать любые. Лучше Qwen4 Embedding 4b\8b. OpenAI - это только формат API, предоставляемый практически всеми сервисами

Пример использования OpenRouter для Embedding:
OPENAI_API_BASE=https://openrouter.ai/api
OPENAI_API_KEY=ваш-ключ-openrouter
OPENAI_MODEL=qwen/qwen3-embedding-8b

Пример запуска с OpenRouter:

docker run -d -e RESET_CACHE=false -e RESET_DATABASE=false -e USESSE=false -e LICENSE_KEY=5a98417f-de9f-468c-b62c-d940a5681da8 -e 1C_BIN_PATH=/1c_docs -e OPENAI_API_BASE="https://openrouter.ai/api" -e OPENAI_API_KEY="ваш-ключ-openrouter" -e OPENAI_MODEL="qwen/qwen3-embedding-8b" -p 8003:8003 -v "C:\Program Files (x86)\1cv8\8.3.27.1936\bin:/1c_docs" -v "E:/bases/mcp_docs:/app/chroma_db" comol/1c_help_mcp:latest

При использовании кастомной модели рекомендую подключить /app/model_cache как том.

На Linux и в Windows 11 если видеокарта Nvidia можно добавить параметр
--gpus all
Есть вероятность что контейнер начнёт использовать видеокарту.
